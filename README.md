# I2C_Protocol_SystemVerilog

SystemVerilog-based implementation of the I2C protocol with a complete RTL design and transaction-level verification environment.

---

## 📡 What is I2C?

**I2C (Inter-Integrated Circuit)** is a **2-wire serial communication protocol** used to connect multiple devices (like sensors, EEPROMs, ADCs) using minimal wiring.

- Developed by **Philips (now NXP)**
- Uses only:
  - **SDA (Serial Data Line)**
  - **SCL (Serial Clock Line)**

---

## 🧠 Basic Idea 

- One device acts as **Master (controller)**
- Other devices act as **Slaves (responders)**
- Master initiates communication and controls timing
- Slave only responds when addressed

---

## 🔑 Key Features

- ⭐ **2-wire communication (SDA + SCL)**
- ⭐ **Multi-master, multi-slave support**
- ⭐ **Address-based communication**
- ⭐ **Half-duplex (one direction at a time)**
- ⭐ **Synchronous (clock-driven)**
- ⭐ **Simple and widely used in embedded systems**

---

## ⚙️ Signals

| Signal | Description |
|--------|------------|
| SDA    | Data line (bidirectional) |
| SCL    | Clock line (generated by master) |

---

## 🔄 Communication Sequence

1. **START Condition**
2. **Slave Address + R/W bit**
3. **ACK from Slave**
4. **Data Transfer (Read/Write)**
5. **ACK/NACK after each byte**
6. **STOP Condition**

---

## ⏱️ Important Conditions

### 🔹 Start Condition
- SDA goes **HIGH → LOW** while SCL is HIGH

### 🔹 Stop Condition
- SDA goes **LOW → HIGH** while SCL is HIGH

---

## 🔢 Addressing

- Typically **7-bit address**
- 8th bit = **R/W bit**
  - `0 → Write`
  - `1 → Read`

---

## 🤝 ACK / NACK

- After every byte transfer:
  - **ACK (0)** → Data received successfully
  - **NACK (1)** → Error / End of communication

⚠️ Important: ACK is **active LOW**

---

## 🔌 Open-Drain Concept (Very Important)

- Devices **cannot drive HIGH**
- They can only:
  - Pull line **LOW (0)**
  - Or **release (Z)**

👉 External **pull-up resistor** makes line HIGH

---

## 🔁 Data Transfer

- Data is sent **MSB (Most Significant Bit) first**
- Each byte = **8 bits**
- Followed by **1 ACK/NACK bit**

---

## 🧱 Architecture

- **Master**
  - Generates clock
  - Controls communication
- **Slave**
  - Responds to address
  - Sends/receives data

---

## ⚡ Speed Modes

| Mode | Speed |
|------|------|
| Standard Mode | 100 kbps |
| Fast Mode     | 400 kbps |
| Fast Mode+    | 1 Mbps |
| High Speed    | 3.4 Mbps |

---

## 🧠 Key Concepts 

- ⭐ I2C uses **open-drain/open-collector** lines  
- ⭐ Requires **pull-up resistors** on SDA & SCL  
- ⭐ **ACK = 0**, **NACK = 1** (very important)  
- ⭐ **Start/Stop conditions depend on SDA transition when SCL is HIGH**  
- ⭐ **Master always controls clock (SCL)**  
- ⭐ Supports **multi-master arbitration**  
- ⭐ Uses **7-bit or 10-bit addressing**  
- ⭐ **Data valid only when SCL is HIGH**  
- ⭐ **Change data only when SCL is LOW**  
- ⭐ **Half-duplex communication**  
- ⭐ Uses **FSM for hardware implementation**  
- ⭐ **Clock stretching** possible by slave  
- ⭐ **Inout signal (SDA)** requires tri-state logic  

---

## 🎯 Why I2C is Used

- Fewer wires → simpler PCB design  
- Easy to connect multiple devices  
- Widely supported in embedded systems  

---

## 🧩 I2C vs SPI (Quick Difference)

| Feature | I2C | SPI |
|--------|-----|-----|
| Wires  | 2   | 4   |
| Speed  | Slower | Faster |
| Addressing | Yes | No |
| Complexity | Simple | Moderate |

---

## 📌 Summary

I2C is a **simple, reliable, and widely used communication protocol** ideal for short-distance, low-speed communication between multiple devices using minimal wiring.
