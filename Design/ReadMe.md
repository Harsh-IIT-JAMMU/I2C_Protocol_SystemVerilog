# 📡 I2C Protocol Design & Verification (SystemVerilog)

This project implements the **I2C (Inter-Integrated Circuit)** protocol using **SystemVerilog**, covering both **RTL design (Master + Slave)** and a structured architecture. It demonstrates communication using only **two wires: SDA (data) and SCL (clock)**.

---

## 🚀 Features

- ✅ I2C Master RTL (Start, Stop, Read, Write)
- ✅ I2C Slave RTL with internal memory (128 x 8-bit)
- ✅ Bidirectional SDA using tri-state logic
- ✅ ACK/NACK handling
- ✅ Parameterized clock (40 MHz → 100 kHz)
- ✅ FSM-based design for clarity and control
- ✅ Modular and scalable architecture

---

## 🧠 How It Works (Layman View)

- Master = "Boss" → initiates communication  
- Slave = "Listener" → responds  
- SDA = carries data  
- SCL = controls timing  

### Communication Flow

1. Start condition  
2. Address transmission  
3. ACK from slave  
4. Data transfer (Read/Write)  
5. ACK/NACK  
6. Stop condition

## ⚙️ Key Design Blocks

### 🔹 I2C Master
- Generates SCL clock  
- Sends START and STOP  
- Sends address + R/W bit  
- Handles read/write operations  
- Checks ACK from slave  

### 🔹 I2C Slave
- Detects START condition  
- Reads address from master  
- Sends ACK  
- Performs:
  - Write → store data in memory  
  - Read → send data from memory  

---

## ⏱️ Timing Details

- System Clock: **40 MHz**  
- I2C Clock: **100 kHz**  
- Each I2C bit divided into **4 phases (pulse 0–3)**  
- Ensures proper synchronization of SDA with SCL  

---

## 🔌 Important Signals

| Signal | Description |
|--------|------------|
| SDA    | Data line (bidirectional) |
| SCL    | Clock generated by master |
| newd   | Start new transaction |
| op     | Operation (0 = write, 1 = read) |
| busy   | Master is active |
| done   | Transaction completed |
| ack_err| Acknowledgment error flag |

## 🧠 Key Concepts

- ⭐ **I2C is Half-Duplex** → Data flows in one direction at a time  
- ⭐ **Open-Drain Concept** → Devices can only pull line LOW, HIGH comes from pull-up resistor  
- ⭐ **Tri-State Logic (Z)** → Used when device releases the bus  
- ⭐ **ACK = 0, NACK = 1** → Counterintuitive but important  
- ⭐ **Start Condition** → SDA falls while SCL is HIGH  
- ⭐ **Stop Condition** → SDA rises while SCL is HIGH  
- ⭐ **MSB First Transmission** → Data is sent from most significant bit  
- ⭐ **Clock Synchronization** → Controlled by master  
- ⭐ **FSM (Finite State Machine)** → Core of protocol implementation  
- ⭐ **Bit-Level Timing Control** → Using pulse (0–3) phases  
- ⭐ **Address + R/W Bit** → Last bit defines read/write  
- ⭐ **Inout Port Usage** → Required for bidirectional SDA  
- ⭐ **Shift Register Logic** → Used to send/receive serial data  
- ⭐ **Memory Mapping in Slave** → Acts like register bank  
- ⭐ **Protocol Compliance** → Correct timing + ACK ensures valid communication
